История языка Java

Создатель языка Java

Язык программирования Java был разработан в американской компании Sun Microsystem, основанной в 1982 году, занимающейся разработкой и продажей технологий в областях телекоммуникации, компьютерной техники и программного обеспечения. Разработал язык программист Джеймс Артур Гослинг (James Arthur Gosling), родившийся 19 мая 1955 года в Канадском городе Калгари. В 1977 он окончил Университет Калгари, во время учёбы в котором он помог с созданием программы для обработки данных со спутника ISIS 2 (International Satellites for Ionospheric Studies). В 1983 Джеймс закончил университет Карнеги-Меллона (Carnegie Mellon University), где получил докторскую степень за защиту диссертации на тему «Алгебраическая манипуляция ограничениями» (в оригинале: The Algebraic Manipulation of Constraints). До этого, в 1981 году, Гослинг с помощью языка C создал Gosling Emacs – первую вариацию текстового редактора Emacs, функционирующую на операционной системе Unix. В 1984 году Джеймс устраивается на работу в Sun Microsystem, где в 1986, совместно с Дэвидом Стюарт Холмс Розенталем (David Stuart Holmes Rosenthal) разработал оконную систему, то есть программное обеспечение, способное к одновременному раздельному управлению различными частями экрана, NeWS (Network extensible Window System). В 2007 году был награждён Орденом Канады (Order of Canada), высшей гражданской наградой Канады, за вклад, внесённый в развитие компьютерных наук, а в 2015 году – Медалью Джона фон Неймана (Institute of Electrical and Electronics Engineers John von Neumann Medal).

Причины разработки языка Java

Создание языка программирования Java напрямую связано с проектом Green, который был начат Sun Microsystem в 1991 году, ведь Java был создан Гослингом именно для реализации данного проекта. Основной задачей проекта стало создание устройства, которое позволило бы компании закрепиться на рынке бытовой техники. Проект возглавляли Патрик Нотон (Patrick Naughton), Майк Шеридан (Mike Sheridan) и Джеймс Гослинг. Последний утверждал, что происходит постоянное улучшение электронного и цифрового наполнения бытовой техники, то есть по пути компьютеризации электронных домовых приборов. В связи с данным предположением, было решено разработать устройство, с помощью которого пользователь мог бы дистанционно управлять окружающей бытовой техникой. В данной разработке особенно важным элементом был язык программирования, который позволял бы запускать необходимое программное обеспечение практически на любом процессоре, и при этом иметь возможность объединять все устройства в единую сеть. Изначально на роль языка для разработки программного обеспечения для устройства был выбран C++, но вскоре стало понятно, что, используя этот язык, поставленную задачу невозможно выполнить, так как C++ не обладал достаточным уровнем портативности и совместимости созданного на нём программного обеспечения. В связи с чем Джеймс Гослинг начал разработку нового языка программирования, и в 1992 создал язык Oak, в дальнейшем, в именно в 1995 году, сменивший название на Java. Но получившаяся разработка не обрела коммерческого успеха, из-за ранней стадии развития отрасли, а также дороговизны устройства. Тогда, из-за всё нарастающей популярности интернета, а точнее «Всемирной паутины» (World Wide Web), в 1993 году командой разработчиков проекта Green было решено создать браузер, который, используя возможности предназначенного для сетевой работы языка Oak, позволил бы увеличить безопасность передачи данных между сервером и клиентами, а также поддерживающий возможность создания любого интерфейса веб-страницы, и использование на ней анимации. Дальнейший рост популярности и востребованности языка Java основан именно на его использовании в данном браузере, имевшим название WebRunner, переименованный в HotJava.

Языки предшественники

Языком, оказавшим наибольшее влияние на Java, является C++, созданный в 1980 году для личных нужд сотрудником компании Bell Labs Бьёрном Страуструпом (Bjarne Stroustrup) на основе языка программирования C, и называющийся на тот момент «C с классами» («C with Classes»). Влияние C++ сразу же видно в синтаксисе Java. Визуально код на языке Java похож на код языка C, но не всегда. Причиной этого является желание Гослинга сделать код Java как можно более понятным для программистов, работающих с языками C и C++. При этом различий между языками довольно много. C++ является компилируемым языком программирования, то есть его программа во время выполнения преобразуется программой компилятором в машинный код, выполняемый компьютером. Программа на языке Java компилируется в специальный байт-код, который в дальнейшем передается виртуальной машине Java (Java Virtual Machine). Данное программное производит интерпретацию байт-кода программы, а именно перевод программы в машинный код и её построчное выполнение, при этом оно поддерживается на всех основных операционных системах, что позволяет один из основных принципов языка Java: «Написал однажды, запускаешь везде» (Write once, run anywhere), так как JVM любого устройства сможет выполнить переданную ей программу. Также Java, в отличии от C++, является полностью объектно-ориентированным языком программирования. Это означает, что основными элементами языка являются: объекты – структуры, которые имеют определённые свойства и способы взаимодействия с собой, методы – функции, описывающие работу с определенными объектами, классы – описание свойств однотипных объектов и методов работы с ними. Написать программу на языке Java без использования этих элементов невозможно, но в C++ это возможно, так как он является объектно-ориентированным языком только частично, многое сохранив от процедурного, то есть основанного на отдельных операторах и командах, которые можно собирать в независимые подпрограммы - процедуры, языка C. Остальные отличия, такие как отсутствие в Java указателей, невозможность использовать системные библиотеки и жёсткая стандартизация всех элементов языка, уже не являются настолько значимыми, как вышеуказанные.

Развитие языка Java

Первая версия Java – Java 1.0 – вышла в 1996 году в составе набора разработчика Java, или же JDK (Java Development Kit), версии 1.0. JDK – программное обеспечение, которое позволяет вести разработку приложений и апплетов, программ, запускающихся в браузере, состоящее из двух основных элементов: инструменты разработки Java (Java Development Tools), включающие в себя инструменты для разработки приложения, такие как компилятор написанной программы в байт-код, и среду выполнения Java (Java Runtime Environment), которая используется для запуска приложений и содержит виртуальную машину Java и необходимые для работы приложений Java библиотеки. JDK в дальнейшем всегда будет иметь ту же версию, что и язык. Через год выходит версия Java 1.1, которая, совместно с версией 1.0, называют Java 1. В версии 1.1 к основе языка из версии 1.0 были сделаны несколько добавлений. В ней была улучшена библиотека для создания графического интерфейса – Abstract Window Toolkit. В данной библиотеке была изменена система событий (AWT Event Model). В версии 1.0 система событий построена на наследовании. Это означает, что если пользователь совершит какое-то действие над интерфейсом, то оно поступит к связанному с этим событием методом, и он либо выполнит это действие, либо передаст его выше по иерархическому дереву. У данной системы есть множество недостатков, таких как необходимость создавать собственный подкласс для каждого события, сложность в использовании разных типов событий, из-за общности связанных с ними методов, и отсутствие фильтрации событий, что приводит к обращению даже к тем компонентам интерфейса, которые не могут обработать данное событие. Для решения данной проблемы в версии 1.1 система наследования была заменена на систему делегации (Delegation Model). Она основана на разделении всех событий на различные классы, для каждого из которых определены также различные методы выполнения событий. Данная система заметно проще предыдущей, и лишена многих её недостатков. Также в версии 1.1 была добавлена рефлексия (Reflection), возможность программы анализировать саму себя во время исполнения и ограниченно манипулировать своими компонентами, например, выводить на экран названия всех объектов и методов того или иного класса, и интерфейс подключения к базе данных Java (The Java Database Connectivity Application Programming Interface), который даёт возможность подключать к программе информацию из внешних баз данных.

В 1998 вышла версия Java 1.2, названная Java 2, вместе с которой были внесены изменения не только в язык программирования, но и формат выпуска языка. JDK был заменён на Java 2 Platform Special Edition, который состоял из самого стандартного издания J2SE, которая содержала в себе инструменты разработки портативных приложений для компьютеров и серверов, корпоративное издание Java 2 Platform Enterprise Edition, содержащее инструменты для создания крупномасштабных программ, и Java 2 Platform Micro Edition, содержащее средства создания приложений для мобильных устройств. В самом языке, основным представленным изменением стал новый инструмент создания графического интерфейса – Swing Graphical User Interface, который пришёл на смену AWT из прошлой версии.

Следующая версия Java 1.3 вышла в 2000 году. В ней была представлена новая виртуальная машина - Java HotSpot Virtual Machine, основной задачей которой являлась оптимизация выполнения программы. Ещё был введён интерфейс вызова удаленных методов (Remote Method Invocation), позволяющий использовать методы объекта, хранящегося на другом устройстве, интерфейс доступа к службам имен и каталогов (Java Naming and Directory Interface), использующийся в разработке больших коммерческих проектов и позволяющий систематизировать хранение множества объектов приложения, а также облегчить дальнейшую работу с ними.

Версия Java 1.4 была выпущена в 2002 году и ввела поддержку нового интернет протокола IPv6, асинхронную или не блокируемую систему ввода и вывода, отличительной особенностью которой является продолжение выполнения заданного процесса во время вывода или ввода данных, что невозможно в системе с блокирующем вводом и выводом, потому что в ней выполняемы процесс будет приостановлен, до полного получения или отправки данных, интерфейс системы логирования (Java Logging API), то есть записи событий, происходящих с объектом, интерфейс ввода/вывода изображений (Image Input/Output API), позволяющий воздействовать на изображения с помощью кода Java.

Вместе с выходом следующей версии языка в 2004 году была изменена система именования версий, но изменение было не принципиальным. Изменилась форма записи номера версии, и теперь версия 1.5 носила номер 5.0. Впоследствии из номера убрали «.0», поэтому следующие версии имели названия Java 6, Java 7 и так далее. В самой Java 5.0 в язык добавили массу изменений, среди которых обобщения (Generics), добавляющие возможность создавать обобщённые классы, интерфейсы и типы, позволяющие использовать в них не заранее заданный тип данных, а определяемый при выполнении программы, аннотации (Annotations), то есть дополнительную информацию, которая не является частью программы, но несёт различную необходимую информацию о самой программе. Была добавлена система авто упаковки и распаковки, которая автоматически изменяет примитивные типы данных, такие как int, char и float, в соответствующие им объектные или типы, то есть в полноценные классы, с возможностью воздействия на них соответствующими методами, что невозможно для примитивных типов, и наоборот – объектные превращает в примитивные. Это сделано с целью упрощения работы с кодом программы.

Последней версией языка Java, выпущенной компанией Sun Microsystem, стала версия Java 6, выпущенная в 2006. В 2009 году американская корпорация Oracle поглотила Sun Microsystem, и дальнейшие версии языка Java выпускались уже ей. Сама версия не содержала новых функциональных изменений языка, а состояла из улучшения работоспособности языка, такие как улучшение работы компилятора, пользовательских интерфейсов разработки, системы подключения к базам данных и использования языком памяти устройства.

Седьмая версия языка Java вышла только в 2011 году. В новой версии было добавлены некоторые синтаксические функции языка, но, как и в прошлой версии, основные нововведения были внесены в платформу языка, то есть в комплект программного обеспечения, обеспечивающего возможность разработки и использования программ и приложений на данном языке. Среди этих нововведений было добавление поддержки виртуальной машиной Java динамических языков программирования, то есть возможность использовать в программе код языков программирования, которые позволяют определять типы данных переменных и функций, при выполнении программы, такие как PHP, Python, Smalltalk и JavaScript. Сам же язык программирования Java относиться к статичным языкам программирования, то есть типы данных в нём строго типизированы, и не могут быть изменены после определения переменной. Также была добавлена система сжатия указателей, что позволило уменьшить количество памяти, выделяемой на их хранение.

Последние крупные изменения в язык были внесены в версиях Java 8 и Java 9. После них, в основном, в язык вносились незначительные изменения, а также улучшения уже существующих возможностей языка. В восьмой версии языка была добавлена поддержка лямбда-выражений, являющимися безымянными классами и методами, которые позволяют упростить вызов и передачу параметров в другие методы. Другим важным добавлением стало добавление интерфейса Stream API, который больше инструментов для работы с коллекциями, то-есть со списками и массивами. Эти добавления вводят в Java элементы функционального программирования, то есть парадигмы программирования, основанной на математических функциях, которая часто противопоставляется объектно-ориентированному подходу к написанию программ. В 9 версии была добавлена поддержка протоколов передачи данных HTTP/2 и WebSocket, которые превосходили протокол HTTP 1.1, используемый в Java до этого. Вместе с поддержкой новых протоколов был добавлен новый интерфейс HTTP client API для работы с новыми протоколами. Другим нововведением стала возможность выполнять отдельно небольшой код языка Java и моментально получить результат без использования новых классов, благодаря утилите, то-есть вспомогательной программе, Jshell. Эта оболочка реализует в языке подход к организации интерактивной среды программирования REPL (Read-eval-print loop), который реализует в языке интерфейс командной строки. Последним важным элементом версии 9 стала система модулей Jigsaw, которая должна была появиться в языке ещё в версии 7. Эта система позволяет разбивать программу на модули, в зависимости от функционала. Реализация данной системы сильно изменяет структуру программы, а также позволяет более тонко настраивать иерархию и взаимодействие различных объектов.
